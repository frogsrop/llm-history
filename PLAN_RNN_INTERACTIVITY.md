# Plan: RNN Module Interactivity Improvements

## Цель
Сделать модуль RNN/LSTM визуально объяснительным — анимация не фоновая, а синхронизированная с реальной генерацией.
Зритель должен видеть: какое слово пришло (x_t), какой контекст передался (h_{t-1}), что предсказано.

---

## Текущее состояние (проблемы)
- CSS `animation: fp` — бесконечное пульсирование, никак не связано с генерацией
- Unrolled diagram (нижняя часть SVG) статичен, показывает первые 5 слов корпуса
- Генерация: токены появляются в тексте, но SVG не реагирует
- Expanded cell (верхняя часть) — красивая, но мертвая

---

## Идеи и приоритеты

### ★★★ Идея 1 — Живая пошаговая анимация unrolled diagram при генерации
**Описание:** При нажатии "Генерировать" нижние ячейки SVG анимируются шаг за шагом:
- Ячейка h_i подсвечивается (highlight stroke)
- Стрелка x_t снизу "прилетает" (stroke-dashoffset анимация)
- Стрелка h_{t-1}→h_t "прилетает" вправо
- Слово под ячейкой появляется — совпадает с токеном в result box
- Следующий шаг начинается после паузы

**Что нужно изменить:**
- HTML: добавить `id` ко всем ячейкам, стрелкам, token labels в unrolled части
- JS: функция `animateGeneration(tokens)` — пошагово управляет SVG-элементами
- CSS: классы `.cell-active`, `.cell-done`, `.arrow-flowing` вместо глобального `.anim`
- Убрать infinite CSS animation с h-стрелок в unrolled части (оставить только в expanded cell)

**Результат:** зритель видит "бегущий" контекст h по цепочке + слово, входящее в сеть.

---

### ★★★ Идея 2 — Динамически расширяемый unrolled diagram (sliding window)
**Описание:** Вместо фиксированных 5 ячеек — генерировать SVG-ячейки динамически по ходу генерации. Показывать последние N ячеек (например 6), остальные уходят влево.

**Что нужно изменить:**
- Заменить статический SVG unrolled-части на JavaScript-генерируемый SVG (или canvas)
- Функция `buildUnrolledSVG(tokens, activeIdx)` — рисует ячейки для заданного набора токенов
- При каждом шаге: добавляем ячейку справа, старые затухают/уходят
- Токены под ячейками = реальные сгенерированные слова

**Ограничение:** SVG в HTML удобнее анимировать, чем canvas. Можно сделать на `<svg>` с JS manipulation.

**Результат:** видна "развёртка" сети в реальном времени, длина == количеству сгенерированных слов.

---

### ★★☆ Идея 3 — Анимация потока в expanded cell
**Описание:** Expanded cell (верхняя часть SVG) тоже реагирует на генерацию:
- При каждом шаге: x_t стрелка "вспыхивает" с текущим словом
- ⊕ → tanh → h_t: анимация "прохождения" сигнала по клетке
- h_t стрелка вправо вспыхивает, показывая передачу контекста

**Что нужно изменить:**
- Добавить id к ключевым элементам expanded cell
- JS: `animateCellStep(word_in, word_out)` — подсвечивает элементы последовательно
- Совместить по тайммингу с Идеей 1

**Результат:** Верхняя диаграмма "оживает" синхронно с нижней.

---

### ★★☆ Идея 4 — Разделение: "Архитектура" и "Генерация шаг за шагом"
**Описание:** Переработать layout модуля:
```
[card: Архитектура — expanded cell, статичный, всегда виден]
[card: Живая генерация — unrolled diagram + controls + result]
```
В "Живой генерации" — слайдер скорости (delay между шагами), кнопка "Пауза/Продолжить".

**Что нужно изменить:**
- Переместить unrolled SVG из карточки архитектуры в карточку генерации
- Добавить слайдер `Скорость анимации` (уже есть `window.animDelay` в utils.js)
- Кнопка паузы (опционально)

### ★☆☆ Идея 5 — Endpoint /api/rnn/generate/stream (SSE)
**Описание:** Потоковая генерация — backend отдает токены по одному через Server-Sent Events.

**Что нужно изменить:**
- `routers/rnn.py`: добавить `EventSourceResponse` endpoint
- `module-3-rnn-lstm.html`: EventSource вместо apiFetch

**Зачем:** Для длинных генераций (10+ слов) — более реалистично, нет задержки на весь fetch.
**Сложность:** Требует `sse-starlette` библиотеки, изменение backend.
**Вывод:** Низкий приоритет — у нас генерация быстрая, fetch достаточен.

## Технические заметки

### SVG анимация — рекомендуемый подход
Использовать JS манипуляцию атрибутами через `classList`, не CSS анимацию.
Для стрелок — `stroke-dasharray` / `stroke-dashoffset` с CSS transition:
```css
.arrow-resting { stroke-dasharray: 80; stroke-dashoffset: 80; }
.arrow-active  { stroke-dashoffset: 0; transition: stroke-dashoffset 0.3s ease; }
```

### Скорость анимации
`window.animDelay` уже используется в utils.js. Добавить слайдер в UI, связать с переменной.

### Совместимость с LSTM режимом
`animateGeneration` должна знать текущий режим (rnn/lstm) и переключать нужный SVG.

### Ограничение на 5 ячеек
Если генерируется > 5 токенов, можно:
- Вариант A: прокрутить SVG (overflow-x: auto)
- Вариант B: показывать sliding window из 5 последних
- Вариант C: динамически добавлять ячейки (Идея 2, сложнее)

Рекомендация: **Вариант B** для первой итерации, затем Вариант C.

## Ожидаемый результат
Зритель на проекторе видит:
1. Клетка слева: "кот" входит снизу (x_t), старый контекст приходит слева (h_{t-1})
2. Клетка "обрабатывает" — tanh вспыхивает
3. Новый h_t уходит вправо — следующая клетка принимает его
4. Под клеткой появляется предсказанное слово
5. Цепочка растёт, пока не накопятся все слова
