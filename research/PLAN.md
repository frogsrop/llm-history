# План: Интерактивное объяснение эволюции LLM

## Что строим

Веб-приложение на FastAPI + HTML/JS, которое объясняет эволюцию языковых моделей.
Через все страницы проходит **единый корпус из 10 предложений** (тема: кот и дождь) —
каждый подход "обучается" на одном и том же тексте, разница в качестве видна наглядно.
Запускается локально (`uvicorn main:app`), открывается в браузере.

**Целевая аудитория: сотрудники ИТ-компании без ML-фона** — HR, бухгалтеры, художники, программисты.
Докладчик: фуллстек-разработчик с опытом обучения ИИ, объясняет аналогии устно.

**Роль страницы: визуальный помощник докладчика**, не самостоятельный учебник.
Формат показа: докладчик шарит экран / проектор — один управляет, все смотрят.

- Минимум текста — только названия и подписи
- Рядом с каждым термином кнопка `?` — клик раскрывает аналогию/объяснение
- Крупные элементы управления, хорошо читается на проекторе
- Горячие клавиши для докладчика: `→` следующий модуль, `Space` запустить анимацию
- Мобильная адаптивность не нужна

## Единый обучающий корпус

**Концепция:** 10 предложений (тема: кот и дождь), на которых "обучается" каждый модуль.
Одинаковый текст → разница в качестве между подходами очевидна.

**Выбранный корпус** (`data/corpus.txt`):
```
кот сидел у окна и смотрел на дождь
дождь шёл по улице и кот не выходил
кот лежал у тёплой печи и слушал дождь
дождь барабанил по крыше и кот дремал
кот дремал у печи пока дождь не прошёл
дождь прошёл и кот прыгнул на окно
на окне кот смотрел на улицу и щурился
солнце вышло и кот лёг на тёплую крышу
кот дремал на крыше пока не пришёл вечер
вечером кот слез с крыши и лёг у печи
```

**Почему этот корпус подходит:**
- Повторяющиеся слова `кот`, `дождь`, `дремал` → N-gram строит нетривиальные таблицы
- Дальние зависимости (начало ↔ конец предложения) → RNN «забывает»
- Многозначные контексты → наглядность для Word2Vec

**Как используется в каждом модуле:**

| Модуль      | Что делает с корпусом                                           |
|-------------|----------------------------------------------------------------|
| N-gram      | Строит таблицу переходов, генерирует продолжение               |
| RNN/LSTM    | PyTorch-модель генерирует продолжение, слайдер hidden size      |
| Word2Vec    | Арифметика векторов на словах корпуса + кураторский VOCAB      |
| LLM-эра     | Attention веса на предложениях корпуса + генерация ruGPT-3 XL  |
| Compare     | N-gram / RNN / ruGPT-3 XL дополняют одно предложение рядом     |

Файл: `corpus.py` читает `data/corpus.txt` — меняем txt, все роутеры автоматически берут новый корпус.

### Шаг 7 — `module-4-llm-era.html` (LLM-эра: конструктор улучшений)
Каждый тумблер **добавляет новый визуальный блок** — накопительный режим: тумблер можно только включить, выключить нельзя. Кнопка "Сбросить всё" возвращает страницу к базовому состоянию (только Seq2Seq).

- [ ] Базовый уровень (всегда виден): **Seq2Seq**
  - Анимация: токены корпуса → энкодер → один вектор (bottleneck) → декодер
  - Визуально видно что вся информация сжата в одну точку — проблема очевидна
- [ ] Тумблер "Attention": появляется новый блок
  - Реальная матрица весов внимания из ruGPT-3 XL на нашем корпусе
  - `routers/llm_era.py`: `GET /api/llm-era/attention?sentence=...` → матрица весов: слой 24 из 48, усреднение по всем 16 головам → `[seq_len × seq_len]`
  - Тепловая карта: видно какие слова "смотрят" друг на друга
- [ ] Transformer: **рассказывается устно** — параллельность и multi-head, без визуального блока
- [ ] Тумблер "Масштаб (LLM)": появляется новый блок
  - Реальная генерация через ruGPT-3 XL (~5GB, ~10GB VRAM, HuggingFace: ai-forever/rugpt3xl)
  - `POST /api/llm-era/generate` → продолжение корпуса от LLM
  - Attention веса тоже берём из ruGPT-3 XL — одна модель для всего
  - Шкала параметров: 117M → 1.3B → 175B → ~1T — визуальное ощущение разницы
  - Слайдер температуры: debounce 1 сек → запрос генерации после паузы

## Технический стек

**Бэкенд:**
- Python + FastAPI — сервер и API
- gensim — загрузка fastText русской модели, cosine similarity
- numpy — векторная арифметика
- Jinja2 — рендеринг HTML-шаблонов

**Фронтенд:**
- Чистый HTML/CSS/JS — без фреймворков
- Canvas API — тепловые карты, 2D-карта векторов
- SVG — схемы архитектур с анимацией
- CSS Transitions/Animations — плавные переходы
- localStorage — сохранение прогресса между сессиями
- fetch API — запросы к бэкенду

**Запуск:** `uvicorn main:app --reload` → `http://localhost:8000`
